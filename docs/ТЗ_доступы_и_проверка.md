## 1. Подключить «Яндекс ID» (нужны ваши доступы) 🔴 СРОЧНОЕ

Фронтенд уже готов: кнопка «Яндекс ID» появится сама, как только провайдер
включится на сервере. От вас — создать OAuth‑приложение в Яндексе:

1. Зайдите на **https://oauth.yandex.ru** → «Создать приложение».
2. Название: `Galathilion`. Платформа: **Веб‑сервисы**.
3. **Redirect URI (Callback)** — впишите ровно:
   ```
   https://galathilion.ru/api/oauth2-redirect
   ```
4. Доступы (scopes): «Доступ к адресу электронной почты» (`login:email`) и
   «Доступ к имени пользователя» (`login:info`). Этого достаточно.
5. После создания скопируйте **ClientID** и **Client secret**.
6. **Пришлите мне ClientID + Client secret** — я включу провайдера в PocketBase
   (Settings → Auth providers → Yandex) и проверю вход end‑to‑end.

> Email/пароль на нашем РФ‑сервере уже работает и законен — Яндекс ID это
> дополнительный удобный способ, не блокер для соответствия 149‑ФЗ.

## 2. Проверка соответствия 149‑ФЗ (после моих правок) ✅ уже должно быть чисто

Что я уже сделал: убрал кнопки Google/Apple, поставил allow‑list (только
Яндекс/VK/банковские OIDC), сторонних auth‑библиотек нет, пароли хранит
PocketBase в хешированном виде, всё под HTTPS.

Проверьте сами (это видит и РКН):
- [ ] Открыть `https://galathilion.ru`, форму входа (бургер → войти), на ней **нет**
      кнопок Google/Apple/Facebook/GitHub/Telegram.
- [ ] Chrome DevTools → **Network** → фильтр XHR/JS при открытии формы входа: **нет**
      обращений к `accounts.google.com`, `appleid.apple.com`, `graph.facebook.com`,
      `github.com/login/oauth`, `discord.com/api/oauth2`, `telegram.org/js/...`.
- [ ] Прогнать сайт через **https://scanbase.ru** — должно быть «соответствует».
- [ ] Проверить ВСЕ места входа, не только главную форму (в нашем SPA вход — одна
      модалка, но убедитесь, что она одна и та же везде).

## 3. Тест авторизации (ручной, в реальном браузере)

- [ ] Регистрация: бургер → «Регистрация» → имя/почта/пароль → создать аккаунт →
      должен залогинить и показать профиль.
- [ ] Выход и повторный вход тем же email/паролем.
- [ ] «Забыли пароль?» — **сейчас покажет ошибку, пока не настроен SMTP** (см. п.4).
- [ ] После Яндекс‑вход (п.1) — кнопка «Яндекс ID» → редирект в Яндекс → возврат и
      вход.

## 4. Письма (SMTP) — нужно для сброса пароля и верификации почты

Без SMTP вход по паролю работает, но «сброс пароля» и «письмо‑подтверждение» —
нет. Чтобы включить:
- [ ] Завести почтовый ящик/SMTP (Яндекс 360 / VK WorkMail / любой РФ‑SMTP).
- [ ] Прислать мне: SMTP‑хост, порт, логин, пароль — я впишу в PocketBase
      (Settings → Mail settings) и проверю отправку.

## 5. Прочее на вашей стороне

- [ ] **Домен:** в ЛК reg.ru статус домена был `UNVERIFIED` — **подтвердите данные

- [ ] **Root‑пароль сервера** был передан в открытом виде — рекомендую сменить и
      перейти на SSH‑ключи (могу подготовить инструкцию).
